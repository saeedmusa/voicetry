# OpenAI Python SDK - Chat Completions API

## Installation

```bash
pip install openai
```

## Basic Setup

```python
from openai import OpenAI

client = OpenAI()
# API key is automatically read from OPENAI_API_KEY environment variable
```

## Chat Completions API Usage

The Chat Completions API is the standard way to generate text from OpenAI models. You can use this API with the `client.chat.completions.create()` method.

### Basic Example

```python
from openai import OpenAI

client = OpenAI()

completion = client.chat.completions.create(
    model="gpt-5.2",
    messages=[
        {"role": "developer", "content": "Talk like a pirate."},
        {
            "role": "user",
            "content": "How do I check if a Python object is an instance of a class?",
        },
    ],
)

print(completion.choices[0].message.content)
```

### Passing Messages to chat.completions.create()

The `messages` parameter is an array of message objects. Each message has a `role` and `content`.

#### Message Format

```python
messages=[
    {
        "role": "developer",
        "content": "You are a helpful assistant."
    },
    {
        "role": "user", 
        "content": "Hello!"
    }
]
```

#### Available Roles

1. **`developer`** - Instructions provided by the application developer, prioritized ahead of user messages. Think of this like a function definition.
2. **`user`** - Instructions provided by an end user, prioritized behind developer messages. Think of this like arguments to a function.
3. **`assistant`** - Messages generated by the model (you typically don't send these, but they appear in responses).

#### Example with Multiple Messages

```python
completion = client.chat.completions.create(
    model="gpt-5.2",
    messages=[
        {
            "role": "developer",
            "content": "You are a coding assistant that explains things simply."
        },
        {
            "role": "user",
            "content": "What is a list in Python?"
        },
        {
            "role": "assistant",
            "content": "A list in Python is a collection of items that can be changed, ordered, and allow duplicates."
        },
        {
            "role": "user",
            "content": "How do I create one?"
        }
    ],
)
```

### Accessing the Response

```python
completion = client.chat.completions.create(
    model="gpt-5.2",
    messages=[{"role": "user", "content": "Say hello"}]
)

# Get the main text content
print(completion.choices[0].message.content)

# Access metadata
print(f"Model: {completion.model}")
print(f"ID: {completion.id}")
print(f"Created: {completion.created}")
```

## Common Parameters

### `model` (required)
The model to use (e.g., `"gpt-5.2"`, `"gpt-4o"`, `"deepseek-chat"` for DeepSeek API)

### `messages` (required)
Array of message objects with `role` and `content`

### `temperature` (optional, default: 1.0)
Controls randomness: 0.0 (deterministic) to 2.0 (more random)

```python
completion = client.chat.completions.create(
    model="gpt-5.2",
    messages=[{"role": "user", "content": "Write a story"}],
    temperature=0.7
)
```

### `max_tokens` (optional)
Maximum number of tokens to generate

```python
completion = client.chat.completions.create(
    model="gpt-5.2",
    messages=[{"role": "user", "content": "Tell me a story"}],
    max_tokens=500
)
```

### `top_p` (optional, default: 1.0)
Nucleus sampling: considers tokens with top_p probability mass

## DeepSeek API Compatibility

To use the OpenAI Python SDK with DeepSeek API:

```python
from openai import OpenAI

client = OpenAI(
    base_url="https://api.deepseek.com",  # DeepSeek API endpoint
    api_key="your-deepseek-api-key"
)

completion = client.chat.completions.create(
    model="deepseek-chat",  # DeepSeek model name
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Hello!"}
    ]
)

print(completion.choices[0].message.content)
```

**Note**: DeepSeek follows the OpenAI chat completions API format exactly, so all the same message structures and parameters work.

## Async Usage

```python
import asyncio
from openai import AsyncOpenAI

async def main():
    client = AsyncOpenAI()
    
    completion = await client.chat.completions.create(
        model="gpt-5.2",
        messages=[{"role": "user", "content": "Say hello"}]
    )
    
    print(completion.choices[0].message.content)

asyncio.run(main())
```
